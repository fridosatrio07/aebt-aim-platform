# Front-End Page Build Spec

## Common Page Rules

All pages must show tenant/project/site context, role/access boundary, data source status, and draft/preliminary guardrails where technical outputs are shown. Use table + drawer/detail pattern for high-volume work. Empty/loading/error/access denied states are required.

| Page | Purpose | Primary users | Page sections | Main components | Table columns where applicable | Filters/saved views | Drawer/detail behavior | State requirements | Mock/API/pending status | Guardrails | Acceptance criteria |
